# EoS AI — Hardware Analysis Guide

## Overview

The EoS AI module in ebuild analyzes hardware designs and generates all
configs needed to build an embedded operating system (eos) and bootloader (eboot).

**No server needed. No API key. No pip install. Just Python 3.9+.**

## Quick Start

```bash
cd ebuild/

# From a simple text prompt
python test_full_pipeline.py --text "STM32H7 with UART SPI I2C CAN WiFi BLE 2MB flash"

# From a hardware design file
python test_full_pipeline.py hardware/board/sample_iot_gateway.yaml

# From any text/YAML/KiCad/BOM file
python test_full_pipeline.py my_board_design.txt
```

## What it generates

From a single hardware description, the pipeline produces:

| Output File | Purpose | Used By |
|---|---|---|
| `board.yaml` | Board definition (MCU, arch, peripherals) | eos build |
| `boot.yaml` | Flash layout, boot policy, signing config | eboot build |
| `eos_product_config.h` | C header with `EOS_ENABLE_*` flags | eos compile |
| `eboot_flash_layout.h` | C header with flash addresses/sizes | eboot compile |

## Complete End-to-End Worked Example

### Input

```
"nRF52840 BLE sensor with I2C temperature and SPI flash, 1MB flash, 256KB RAM"
```

### Step 1: Run the analyzer

```bash
cd ebuild/
python test_full_pipeline.py --text "nRF52840 BLE sensor with I2C temperature and SPI flash, 1MB flash, 256KB RAM"
```

### Step 2: Generated `board.yaml`

```yaml
# Auto-generated by EoS AI Analyzer
board:
  name: nrf52840_ble_sensor
  vendor: nordic
  architecture: arm
  core: cortex-m4f
  clock_mhz: 64

memory:
  flash:
    size_kb: 1024
    base_address: 0x00000000
  ram:
    size_kb: 256
    base_address: 0x20000000

peripherals:
  - type: BLE
    version: "5.0"
    features: [advertising, central, peripheral]
  - type: I2C
    ports: 1
    speed: fast      # 400 kHz
  - type: SPI
    ports: 1
    speed: 8000000   # 8 MHz
  - type: GPIO
    count: 48
  - type: UART
    ports: 1
    baudrate: 115200
```

### Step 3: Generated `boot.yaml`

```yaml
# Auto-generated by EoS AI Analyzer
boot:
  board: nrf52840_ble_sensor
  flash_size_kb: 1024
  policy: signed          # require signed firmware
  signing: ecdsa_p256     # ECDSA P-256 signatures

layout:
  stage0:
    offset: 0x00000
    size_kb: 16
  stage1:
    offset: 0x04000
    size_kb: 64
  bootctl:
    offset: 0x14000
    size_kb: 8           # 2 x 4KB boot control blocks
  slot_a:
    offset: 0x16000
    size_kb: 468
  slot_b:
    offset: 0x8A000
    size_kb: 468
  storage:
    offset: 0xFE000
    size_kb: 8           # config storage
```

### Step 4: Generated `eos_product_config.h`

```c
/**
 * EoS Product Configuration — Auto-generated by EoS AI
 *
 * Board:  nrf52840_ble_sensor
 * MCU:    NRF52840 (ARM Cortex-M4F, 64 MHz)
 * Flash:  1024 KB
 * RAM:    256 KB
 */

#ifndef EOS_PRODUCT_CONFIG_H
#define EOS_PRODUCT_CONFIG_H

#define EOS_PRODUCT_NAME         "nrf52840_ble_sensor"
#define EOS_ARCH                 "arm"
#define EOS_CORE                 "cortex-m4f"
#define EOS_CLOCK_MHZ            64
#define EOS_FLASH_KB             1024
#define EOS_RAM_KB               256

/* --- Enabled peripherals --- */
#define EOS_ENABLE_GPIO          1
#define EOS_ENABLE_UART          1
#define EOS_ENABLE_SPI           1
#define EOS_ENABLE_I2C           1
#define EOS_ENABLE_BLE           1

/* --- Disabled peripherals --- */
#define EOS_ENABLE_ADC           0
#define EOS_ENABLE_DAC           0
#define EOS_ENABLE_PWM           0
#define EOS_ENABLE_CAN           0
#define EOS_ENABLE_USB           0
#define EOS_ENABLE_ETHERNET      0
#define EOS_ENABLE_WIFI          0
#define EOS_ENABLE_CELLULAR      0
#define EOS_ENABLE_NFC           0
#define EOS_ENABLE_CAMERA        0
#define EOS_ENABLE_AUDIO         0
#define EOS_ENABLE_DISPLAY       0
#define EOS_ENABLE_MOTOR         0
#define EOS_ENABLE_GNSS          0
#define EOS_ENABLE_IMU           0
#define EOS_ENABLE_MULTICORE     0

#endif /* EOS_PRODUCT_CONFIG_H */
```

### Step 5: Generated `eboot_flash_layout.h`

```c
/**
 * eboot Flash Layout — Auto-generated by EoS AI
 * Board: nrf52840_ble_sensor (1024 KB flash)
 */

#ifndef EBOOT_FLASH_LAYOUT_H
#define EBOOT_FLASH_LAYOUT_H

#define EBOOT_FLASH_BASE         0x00000000
#define EBOOT_FLASH_SIZE         (1024 * 1024)   /* 1 MB */

#define EBOOT_STAGE0_OFFSET      0x00000
#define EBOOT_STAGE0_SIZE        (16 * 1024)     /* 16 KB */

#define EBOOT_STAGE1_OFFSET      0x04000
#define EBOOT_STAGE1_SIZE        (64 * 1024)     /* 64 KB */

#define EBOOT_BOOTCTL_OFFSET     0x14000
#define EBOOT_BOOTCTL_SIZE       (8 * 1024)      /* 8 KB */

#define EBOOT_SLOT_A_OFFSET      0x16000
#define EBOOT_SLOT_A_SIZE        (468 * 1024)    /* 468 KB */

#define EBOOT_SLOT_B_OFFSET      0x8A000
#define EBOOT_SLOT_B_SIZE        (468 * 1024)    /* 468 KB */

#define EBOOT_STORAGE_OFFSET     0xFE000
#define EBOOT_STORAGE_SIZE       (8 * 1024)      /* 8 KB */

#endif /* EBOOT_FLASH_LAYOUT_H */
```

### Step 6: Build everything

```bash
# Copy generated config into eos
cp _generated/eos_product_config.h ../eos/include/eos/eos_product_config.h

# Build eos firmware
cd ../eos
cmake -B build -DEOS_PRODUCT=iot
cmake --build build

# Build eboot bootloader
cd ../eboot
cmake -B build -DEBLDR_BOARD=nrf52
cmake --build build

# Sign firmware
cd tools
python3 sign_image.py --generate-key sensor-key.pem
python3 sign_image.py --key sensor-key.pem \
  --input ../../eos/build/firmware.bin \
  --output signed-sensor-fw.bin

# Flash
nrfjprog --program ../build/eboot.hex --chiperase
nrfjprog --program signed-sensor-fw.hex --verify
nrfjprog --reset
```
